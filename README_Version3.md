# Platinum Store Pro

تم تهيئة المشروع للعمل محليًا والنشر. ملاحظات مهمة حول مفتاح Gemini API:

أمن
- لا تضع مفتاح API داخل الشيفرة المصدرية في مستودعات عامة.
- استخدم حقن متغيّئات بيئة عند النشر أو ضع المفتاح في ملف محلي لا يتم رفعه (.env) أو ضعه مؤقتًا في وسم meta في index.html.

كيفية التشغيل محليًا
1. تثبيت الحزم:
   npm install

2. تزويد مفتاح Gemini:
   - للطريقة الأسرع محليًا: افتح `index.html` وادخل مفتاحك في وسم الـ meta:
     <meta name="gemini-api-key" content="YOUR_GEMINI_KEY_HERE">
   - أو اجعل سيرفرك يحقن الوسم أعلاه تلقائيًا قبل تقديم الصفحة.

   ملاحظة: لا ترفع المفتاح إلى المستودعات.

3. ت��غيل المشروع:
   npm run dev
   (سيعمل التطبيق على http://localhost:3000 بشكل افتراضي)

بناء ZIP جاهز للنشر (محلي)
- لتجميع المشروع في أرشيف ZIP جاهز للنشر محليًا:
  1) تأكد أنك وضعت المفتاح في index.html (أو أنك ستحقنه في الاستضافة).
  2) شغّل:
     npm install
     npm run build
  3) أنشئ أرشيف ZIP للملفات المطلوبة (من جذر المشروع):
     zip -r platinum-store-pro-ready.zip dist index.html package.json README.md src public -x "node_modules/*" ".git/*"

ملاحظة حول المفتاح الذي زوّدته:
- لم أقم بإدراج المفتاح الذي أرسلتَه في أي ملف (لأسباب أمنية). اتّبع التعليمات أعلاه لإضافته محليًا أو عبر سيرفرك.
